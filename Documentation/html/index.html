    Minimum Requirements

    iOS 4 is minimum supported version by this SDK. 

    Using the SDK

    1. Install the embedded static framework to your application. To do this just drag and drop Socialize-IOS.embeddedframework from project root (i.e. socialize-sdk-ios folder) to your framework section.
    2. Add third party libraries with headers: JSONKit, OAuthConsumer, FacebookSDK to your solution if you did not have them before. These libraries can be found either in the socialize-sdk-ios/Socialize/Libraries/ folder or downloaded from the Internet.

    Note: the correct work of the SDK framework is guaranteed only when you use libraries from the socialize-sdk-ios/Socialize/Libraries/ folder.

    1. Add CoreGraphics.framework to your application target.

    Additional configurations

    1. First of all, you should be sure that Socialize-iOS.framework and resources from the Resources folder were added as the dependence to the correct target.
    2. Add -ObjC flag to the Other Linker Flags in the build settings of your application target. Please use the flag exactly as it is—the most common mistake here tends to be misspelling or incorrect capitalization.

    Building the SDK

    To use the SDK, you should perform the next actions:

    1. Build the SDK using terminal command: make test. The output of this command’s execution will be the embedded static framework.

    Note: make test command also builds the distribution version of the Sample application.

    To get a successful build you have to change the provision credentials in the make file to your own.

    # build sample application

            xcodebuild -target SampleSdkApp -configuration Distribution -sdk iphoneos PROVISIONING_PROFILE="User provision profile" CODE_SIGN_IDENTITY="iPhone Distribution: user identity" build

    If you don’t have a valid Apple provision, change the SDK type to the simulator.

    xcodebuild -target SampleSdkApp -configuration Debug -sdk iphonesimulator build

    Important Note:

    Everything in Socialize revolves around the concept of “entity.” An entity can be liked, unliked, commented on, etc. In Socialize, an entity can only be a URL. So when creating an entity remember to input the key for the entity as a URL otherwise you will get a failure.

    To make it easier for third party developers, the entity is automatically created whenever they are liking, creating a comment or viewing an entity (if it already has not been created through an explicit createEntity method).

    Step 2: Instantiate

    // Allocate memory for the instance

    Socialize socialize = [[Socialize alloc] initWithDelegate:self];

    Step 3: Authentication

    1. Anonymous authentication

    // invoke the call

    [socialize authenticateWithApiKey:@"ApiKey" apiSecret:@"ApiSecret"];

    1. Authentication via Facebook

    To prevent authentication errors while re-authenticating a second time, you have to perform a clean up call before the action.

    //clean up previous authentication information

    [socialize removeAuthenticationInfo];

    // invoke the call if you would like the SDK receive third party access token

    [socialize authenticateWithApiKey:@"Authenticating"

     apiSecret:@"ApiSecret" thirdPartyAppId:@"AppId" thirdPartyName:FacebookAuth];

    // invoke the call if you already have had  the  third party access token

    [socialize authenticateWithApiKey:@"ApiKey" apiSecret:@"ApiSecret" thirdPartyAuthToken: @"ThirdPartyToken" thirdPartyAppId:@"AppId" thirdPartyName:FacebookAuth];

    To complete Facebook authentication, implement in Application delegate the next method:

    // invoke the call

    – (BOOL)application:(UIApplication )application handleOpenURL:(NSURL )url

    {

        return [socialize.authService handleOpenURL: url];

    }

    Finally, to enable SSO support you should change the .plist file that handles configuration for the app. XCode creates this file automatically when the project is created. A specific URL needs to be registered in this file that uniquely identifies the app with iOS. Create a new row named URL types with a single item, URL Schemes, containing a single value, fbYOUR_APP_ID (the literal characters fb followed by your app ID).     Create a Like

    Step 1: Include <Socialize-iOS/Socialize.h> in your implementation files.

    Step 2: For liking

    // Allocate memory for the instance

    Socialize socialize = [[Socialize alloc] initWithDelegate:self];

    // invoke the call (the latitude and the longitude can be nil)

    [socialize likeEntityWithKey:@"www.bbc.co.uk" longitude:[NSNumber numberWithFloat:-37.256] latitude:[NSNumber numberWithFloat:122.452314]

    ];

    Step 3: Implementing the delegate

    //if the like fails this is called

    –(void)service:(SocializeService)service didFail:(NSError)error{

    }

    // if the like is created this is called

    –(void)service:(SocializeService)service didCreate:(id<SocializeObject>object){

        if ([object conformsToProtocol:@protocol(SocializeLike)]){

                id<SocializeLike> like = (id<SocializeLike>)object;

                [like  retain];

            }

    }

    –(void)service:(SocializeService)service didFail:(NSError)error{

    –(void)service:(SocializeService)service didFetch:(NSArray)dataArray{

        if ([dataArray count]){

            id<SocializeObject> object = [dataArray objectAtIndex:0];

            if ([object conformsToProtocol:@protocol(SocializeLike) ]){

                 likes = [dataArray retain];

             }

    }
