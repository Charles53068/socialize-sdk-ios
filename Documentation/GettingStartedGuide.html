Minimum Requirements

iOS 4 is minimum supported version by this SDK. 

Using the SDK

To use the SDK user should perform the next actions:

  1. Build the SDK using terminal command: make test. The output of this command’s execution  will be the embedded static framework.

Note: make test command builds also the distribution version of the Sample application.

To get the success build user have to change provision credentials in the make file to his own.

# build sample application

        xcodebuild -target SampleSdkApp -configuration Distribution -sdk iphoneos PROVISIONING_PROFILE="User provision profile" CODE_SIGN_IDENTITY="iPhone Distribution: user identity" build

if you don’t have a valid apple provision change sdk type to the simulator.

xcodebuild -target SampleSdkApp -configuration Debug -sdk iphonesimulator build

  1. Install the embedded static framework to your application. To do this just drag&drop Socialize-IOS.embeddedframework from build folder to your framework section.
  2. Setup project configuration.
  3. Include <Socialize-IOS/Socialize.h> in your implementation files.
  4. Add third party libraries with headers: JSONKit, OAuthConsumer, FacebookSDK to your solution if you did not have tham before. This libraries could be found either in socialize-sdk-ios/Socialize/Libraries/ folder or downloaded from the Internet.

Note: the correct work of the SDK framework is guaranteed only  when you use libraries from socialize-sdk-ios/Socialize/Libraries/ folder.

  1. Add CoreGraphics.framework to your application target.

        

How to setup your project

  1. First off all you should be sure that Socialize-IOS.framework and resources from Resources folder were added as the dependence to the correct target.
  2. Add -ObjC flag to the Other Linker Flags in build settings of your application target.

Important Note:-

Every thing in socialize revolves around the concept of “Entity”. An entity can be liked, unliked, commented on etc. In Socialize an entity can only be a url. So when creating an entity always remember to input the key for the entity as a url otherwise you will get a failure.

And to make it easier for third party developers the entity is automatically created when ever they are liking, creating a comment or viewing an entity, if it already has not been created through an explicit createEntity method.

Authentication

Where delegate is a pointer to user object that implements SocializeServiceDelegate protocol or nil.

Step 1:- Instantiate

// Allocate memory for the instance

Socialize* socialize = [[Socialize alloc] initWithDelegate:self];

Step 2: -Authentication

  1. Anonymous authentication

// invoke the call

[socialize authenticateWithApiKey:@”ApiKey” apiSecret:@”ApiSecret”];

  1. Authentication via Facebook

To prevent authentication errors while second time re-authentication you have to perform clean up call before action.

//clean up previous authentication information

[socialize removeAuthenticationInfo];

// invoke the call if you would the SDK receive third party access token

[socialize authenticateWithApiKey:@”ApiKey” apiSecret:@”ApiSecret” thirdPartyAppId:@”AppId” thirdPartyName:FacebookAuth];

// invoke the call if you would the SDK receive third party access token

[socialize authenticateWithApiKey:@”ApiKey” apiSecret:@”ApiSecret” thirdPartyAuthToken: @”ThirdPartyToken” thirdPartyAppId:@”AppId” thirdPartyName:FacebookAuth];

To complete Facebook authentication implement in Application delegate the next method:

// invoke the call

- (BOOL)application:(UIApplication *)application handleOpenURL:(NSURL *)url

{

    return [authenticationViewController.socialize.authService handleOpenURL: url];

}

The last thing that needs to be accomplished to enable SSO support is a change to the .plist file that handles configuration for the app. XCode creates this file automatically when the project is created. A specific URL needs to be registered in this file that uniquely identifies the app with iOS. Create a new row named URL types with a single item, URL Schemes, containing a single value, fbYOUR_APP_ID (the literal characters fb followed by your app id). Create a like

Step 1: For liking

// Allocate memory for the instance

Socialize* socialize = [[Socialize alloc] initWithDelegate:self];

// invoke the call (the latitude and the longitude can be nil)

[socialize likeEntityWithKey:@”www.bbc.co.uk” longitude:[NSNumber numberWithFloat:-37.256] latitude:[NSNumber numberWithFloat:122.452314]

];

Step 2: Implementing the delegate

//if the like fails this is called

-(void)service:(SocializeService*)service didFail:(NSError*)error{}

// if the like is created this is called

-(void)service:(SocializeService*)service didCreateWithElements:(NSArray*)dataArray andErrorList:(id)errorList{

        if ([[dataArray objectAtIndex:0]  

                              conformsToProtocol:@protocol(SocializeLike) ]){

           //_like is object level instance of type <SocializeLike>

            _like = [dataArray objectAtIndex:0];

            [_like  retain];

        }

}

Getting the like list

This will be helpful in finding out which users have liked the entity.

Step 1: Getting the list

// Allocate memory for the instance

Socialize* socialize = [[Socialize alloc] initWithDelegate:self];

[socialize [self.socialize getLikesForEntityKey:

url first:[NSNumber numberWithInt:1] last:[NSNumber numberWithInt:10]

];

Step 2: Implementing the delegate

//if the unlike fails this is called

-(void)service:(SocializeService*)service didFail:(NSError*)error{

}

-(void)service:(SocializeService*)service didFetch:(NSArray*)dataArray{

    DLog(@"didFetchElements %@", dataArray);

    if ([dataArray count]){

        id<SocializeObject> object = [dataArray objectAtIndex:0];

        if ([object conformsToProtocol:@protocol(SocializeLike) ]){

             _likes = [dataArray retain];

         }

}
